# Security Policy

## Supported versions

Security fixes are applied to the current `main` branch.

## Reporting a vulnerability

Do not open public GitHub issues for security-sensitive reports.

If you discover a vulnerability in the repo tooling, generated artifacts, or release process:

1. Prepare a short report with impact, reproduction steps, and any suggested mitigation.
2. Send it privately to the maintainers if a private contact channel is available.
3. If no private contact channel is available yet, open a minimal GitHub issue without exploit details and request a private follow-up path.

Please avoid publishing proof-of-concept details until maintainers have had reasonable time to investigate and patch.
