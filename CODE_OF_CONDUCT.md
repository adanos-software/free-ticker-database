# Code of Conduct

## Our standard

This project expects respectful, constructive, and technically honest collaboration.

Examples of expected behavior:

- Assume good intent, but be precise about problems.
- Focus criticism on data, code, and decisions, not people.
- Share evidence when proposing corrections.
- Keep discussions useful for maintainers and downstream users.

Examples of unacceptable behavior:

- Personal attacks, harassment, or abusive language
- Deliberate misinformation or fabricated evidence
- Spam, trolling, or low-effort issue noise
- Publishing private or sensitive information without consent

## Scope

This Code of Conduct applies to issues, pull requests, discussions, and other public project spaces.

## Enforcement

Project maintainers may edit, remove, or close content that violates these standards and may restrict repeat offenders from participating.

## Reporting

If you need to report a serious conduct issue, open a private security-style report as described in [SECURITY.md](SECURITY.md), or contact the maintainers through a non-public channel if one is available.
